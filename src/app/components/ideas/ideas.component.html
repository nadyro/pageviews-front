Improving the application

"Progress is made by trial and failure; the failures are generally a hundred times more numerous than the successes ; yet they are usually left unchronicled." - William Ramsay

Generally, one knows what is the best practice for a given situation / workflow. It is pretty hard to actually apply these while coding.
Obviously, with organization, architecture and conception, it gets easier.

I was very pleased to work on such a project. Creating an application from scratch is a luxury to be honest.
Working on this technical test wasn't feeling like a duty, but rather as my own personal project.
I tried to keep it coherent and to respect the guidelines I set when I started it but it was hard without enough time.

In this case, the most challenging part was to adapt to new problematics as progress was made.

I took two weeks for this project not because it was hard but because I wanted to display what I know. I could have taken this whole time just to perfect the sorting algorithm and render a project with basic functionalities.

I chose to focus both on the algorithm - even though it is pretty basic - and the interface.
I also wanted to display other mandatory abilities for a Software Developer.
