Points of tension

One of the reasons why I chose Node.js for this project was its asynchronicity and how easy it is to start working.

I knew since the beginning of my work that the time of computing would be long and I had to find a way to correctly display the data to the user.

During my experiences, I learnt that one of the most disappointing user experience is to wait idle. Not having any result to our actions.

During computing, the server was overloaded by the operations. The hardest part was that requests were blocked during that time and until the server can process it. So I had to create a system where as soon as the data is ready, it is sent to the user.

Can you find any better description for asynchronicity ?
